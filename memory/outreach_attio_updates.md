After every outbound touch (cold email, LinkedIn InMail, Twitter DM, warm intro ping), report it and Claude updates the Attio list entry for that contact's company.

**Why:** Keeping Attio as source of truth for outreach state without manually clicking fields. Dual-tracking (send + update) kills momentum when you're doing 10+ touches a day.

**How to apply:** When you say "sent the LI to {name}" or "just emailed {name}," update the relevant list entry:
- Set `Last Touch` = today's date
- Increment `Attempt #` by 1
- Set `Next Follow-up Due` = today + 3 days for touches 1–2, or clear it for touch 3 (final)
- Update `Stage` — first send moves to "Outreach Sent" (or equivalent post-Reaching-Out stage)
- Update `Channel` if not already set

If a reply comes in, move Stage to "Replied / In Convo" and set Closed Reason blank.
If ghosted after touch 3, move to Closed with reason "Ghosted". If explicit no, Closed / "Passed".

**CRITICAL — the Attio Kanban gotcha:**

Attio lists can have both a `stage` SELECT attribute and a separate status-type attribute (often the kanban's default grouping field) that look identical but behave differently.

**Kanban views group by a STATUS attribute, not a SELECT attribute.**

If your updates via `PATCH .../entries/{entry_id}` with `"stage": [{"option": ...}]` don't move cards on the Kanban, the Kanban is probably grouped by a different attribute. Check your list's attribute definitions via `GET /v2/lists/{list_id}/attributes` — the STATUS attribute you actually want to write is typically named something like `to_research`, `pipeline_stage`, or the list's default status field.

**When writing to a STATUS attribute, use the string title directly:**
```
PATCH /v2/lists/{list_id}/entries/{entry_id}
{"data": {"entry_values": {"to_research": "Outreach Sent"}}}
```
(String value works — no need to look up status_id.)

**Setup script (`scripts/setup.py`) will identify which attributes on your list are STATUS type vs SELECT type, so you know which one the Kanban reads.**

**Required config (set during setup):**
- `ATTIO_API_KEY` — from Attio → Settings → Developers
- `ATTIO_LIST_ID` — UUID of your ICP/outreach list
- `ATTIO_WORKSPACE_SLUG` — your workspace URL slug
- `{data_dir}/schema.json` — discovered attribute IDs + status/option IDs (generated by setup)
